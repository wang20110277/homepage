请按照以下步骤创建一个全面的 checkpoint 提交：

1. **如需要初始化 Git**：如果项目尚未初始化 git，运行 `git init`。

2. **分析所有变更**：
   - 运行 `git status` 查看所有已跟踪和未跟踪的文件
   - 运行 `git diff` 查看已跟踪文件的详细变更
   - 运行 `git log -5 --oneline` 了解本仓库的提交消息风格

3. **暂存所有内容**：
   - 添加所有已跟踪的变更（修改和删除的文件）
   - 添加所有未跟踪的文件（新文件）
   - 使用 `git add -A` 或 `git add .` 暂存所有内容

4. **创建详细的提交消息（使用中文）**：
   - **第一行**：写一个清晰简洁的摘要（50-72 字符），描述主要变更
     - 使用祈使语气（例如："添加功能" 而不是 "已添加功能"）
     - 示例：
       * "feat: 添加用户认证功能"
       * "fix: 修复数据库连接问题"
       * "refactor: 重构 API 路由结构"
       * "feat: 实现 OCR 文字识别服务"
   - **正文**：提供详细描述，包括：
     - 做了哪些变更（关键修改列表）
     - 为什么做这些变更（目的/动机）
     - 任何重要的技术细节或决策
     - 破坏性变更或迁移说明（如适用）
   - **页脚**：包含 Git 安全协议中的共同作者署名

5. **执行提交**：使用正确格式的消息创建提交，遵循本仓库的约定。

重要提示：
- 不要跳过任何文件 - 包含所有内容
- 提交消息要足够详细，让审查 git log 的人能理解完成了什么
- 遵循项目现有的提交消息约定（先检查 git log）
- 在提交消息中包含 Claude Code 共同作者署名
- **所有提交消息必须使用中文描述**
